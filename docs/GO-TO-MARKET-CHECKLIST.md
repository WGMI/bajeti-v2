# Bajeti V1 Go-To-Market Checklist

Use this as the final pass before inviting the first few users. Keep the launch small, learn fast, and only mark an item done when it is truly ready for someone outside the team.

## 1. Product Readiness

- [ ] Complete one fresh end-to-end signup test on web.
- [ ] Complete one fresh end-to-end signup test on mobile.
- [ ] Confirm login, logout, and email verification work reliably.
- [ ] Add a test transaction manually and confirm it appears in summaries, monthly view, and transaction history.
- [ ] Import a realistic SMS sample and confirm the parsed transaction details are correct.
- [ ] Confirm categories, accounts, transfers, rules, and settings save correctly.
- [ ] Confirm the app handles empty states clearly for a brand-new user.
- [ ] Confirm the app works on at least one Android phone and one desktop browser.
- [ ] Check that all visible text, app name, icon, splash screen, and branding feel launch-ready.
- [ ] Fix or document any known issue that would block a first user from understanding or trusting the app.

## 2. Trust, Privacy, And Safety

- [ ] Write a short plain-language privacy note explaining what financial/SMS data is used and why.
- [ ] Confirm users can understand that their data is private to their account.
- [ ] Confirm production database access, API keys, and environment variables are not exposed.
- [ ] Confirm test users and seed data are removed from production.
- [ ] Confirm backups or export options are understood before inviting real users.
- [ ] Prepare a short answer for: "Is my money data safe?"
- [ ] Prepare a short answer for: "Why do you need SMS access?"

## 3. Positioning

- [ ] Define the primary user for V1 in one sentence.
- [ ] Define the core promise in one sentence.
- [ ] List the top three user pains Bajeti solves.
- [ ] Write a 30-second product pitch.
- [ ] Write a one-paragraph invite message for friends, colleagues, or early testers.
- [ ] Prepare 3-5 screenshots or a short screen recording showing the core workflow.
- [ ] Decide what users should do first after signup.

## 4. Launch Setup

- [ ] Confirm the production URL or app install link is ready to share.
- [ ] Confirm analytics or basic event tracking is working for signup, first transaction, and SMS import.
- [ ] Create a simple feedback form.
- [ ] Create a place to track launch users, status, feedback, and follow-ups.
- [ ] Prepare a support channel such as WhatsApp, email, or a shared inbox.
- [ ] Decide how many first users to invite in the first batch.
- [ ] Decide what counts as a successful first week.

## 5. First-User Outreach

- [ ] Make a list of 20-30 potential early users.
- [ ] Pick the first 5-10 users who are likely to give thoughtful feedback.
- [ ] Send each person a direct personal invite, not a mass message.
- [ ] Ask each user for permission to follow up after they try it.
- [ ] Schedule at least 3 short feedback calls or chats.
- [ ] Track who signed up, who got stuck, who activated, and who gave feedback.

## 6. Onboarding And Activation

- [ ] Confirm the first-run experience makes the next action obvious.
- [ ] Create a simple onboarding script for helping a new user through setup.
- [ ] Check whether users can add or import their first transaction without help.
- [ ] Check whether users understand categories and accounts.
- [ ] Check whether users understand what the dashboard is telling them.
- [ ] Record the top three points where users hesitate or ask questions.

## 7. Feedback Loop

- [ ] Ask every first user what they expected Bajeti to do.
- [ ] Ask every first user what felt confusing.
- [ ] Ask every first user what felt useful enough to keep using.
- [ ] Ask every first user what would make them recommend it.
- [ ] Group feedback into bugs, confusion, missing features, and nice-to-haves.
- [ ] Fix critical bugs before inviting the next batch.
- [ ] Choose only 1-3 improvements for the next release cycle.

## 8. Decision Gate

- [ ] At least 5 users completed signup.
- [ ] At least 3 users added or imported real transactions.
- [ ] At least 3 users gave specific feedback.
- [ ] No critical trust, privacy, or data-loss issue is open.
- [ ] The next batch of users is clearly defined.
- [ ] The next product priorities are written down.

## Simple Success Metrics For V1

- [ ] 10 invited users.
- [ ] 5 completed signups.
- [ ] 3 activated users who add or import real data.
- [ ] 3 feedback conversations.
- [ ] 1 clear reason users say they would keep using Bajeti.
- [ ] 1 clear reason users hesitate or drop off.
